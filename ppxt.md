## AgentMem 商业计划路演提纲（讲稿版）

> 用途：配合 `bpppt.md` 的 PPT 使用，作为讲解要点和补充说明。

---

### 1. 开场（1–2 分钟）
- 先一句话：**“我们在做 AI 时代的 Redis——专门给 AI Agent 提供长期记忆和理解能力。”**
- 补一句：
  - 大模型已经足够聪明，但它们**几乎不记得你是谁、你之前说过什么、公司发生过什么**。
  - 我们要解决的，就是“AI 怎么真正记住并理解世界和用户”。

---

### 2. 问题与机会（3–4 分钟）
- 问题 1：**今天的 AI Agent 没有真正的长期记忆**。
  - 每次对话都从零开始，无法连续理解用户、项目和业务上下文。
  - 企业内部的数据分散在文档、IM、工单、业务系统里，Agent 很难统一理解。
- 问题 2：**现有 RAG/向量库方案很难支撑复杂业务 Agent**。
  - 只是“按相似度检索文档”，做不到复杂的多跳推理和长期演化。
  - 更谈不上对“用户、设备、组织”的持续画像与关系建模。
- 问题 3：**从企业视角看，性能 / 成本 / 安全 / 生态都是大坑**。
  - 高并发下 LLM 调用量爆炸，Token 成本不可控。
  - 海外云 + 第三方向量库在金融、政企里几乎过不了合规。
  - 国内需要一个**真正兼容本土生态（鸿蒙、仓颉、华为 MAAS）的基础设施层**。

> 过渡：所以我们认为，“AI 记忆基础设施”会是继数据库、消息队列之后的下一层基础设施，市场空间非常大，而且现在还处在早期。

---

### 3. 我们的方案：AgentMem 是什么？（3–4 分钟）
- 用一句简单的话：
  - **AgentMem = 面向 AI Agent 的“记忆操作系统” + “高性能记忆数据库”。**
- 三个关键点：
  1. **统一的 Memory API**：
     - 开发者只需要 `add / search / update / delete / clear` 这套 API，就能让任意 Agent 拥有长期记忆。
     - Python SDK 已经提供了 Mem0 兼容接口，迁移成本非常低。
  2. **原生多智能体记忆架构**：
     - 我们内部不是一个大 Memory 类，而是 **8 个专门的记忆 Agent + MetaMemoryManager 调度**。
     - 可以并行处理：事实抽取、向量检索、图推理、聚类、去重等不同任务。
  3. **图 + 向量混合记忆，引擎级性能**：
     - Rust 实现，有完整的 GraphMemoryEngine 与性能压测体系。
     - 对标 mem0 的 1 万 ops/s 级别吞吐，并在多智能体架构下进一步优化。

---

### 4. 技术护城河（4–5 分钟）
- **护城河 1：已经写好的多智能体记忆内核**
  - 文档中可以看到：目前 Rust 代码里已经有完整的多 Agent 架构、MetaMemoryManager 和三种负载均衡策略。
  - 这不是 PPT 上写的“规划”，而是**真实存在的代码，只需要逐步启用和优化**。
- **护城河 2：图 + 向量混合记忆能力**
  - 我们有一整套 GraphMemoryEngine，可以把“人、事件、文档、对话、决策”连成图进行推理。
  - 在多跳对话、复杂关系问答、企业级审计/风控场景里，这是 RAG 很难替代的能力。
- **护城河 3：极致性能 + Rust 基础设施工程能力**
  - 完整的压测与优化文档（包括 CPU 利用率、P95 延迟、批量写入等），说明我们不是“做 Demo”，而是在对标生产级性能。
  - Rust 带来的是**类型安全 + 高性能 + 可预期的资源占用**。
- **护城河 4：本土生态一体化（鸿蒙 + 仓颉 + 华为 MAAS）**
  - 仓库中已经有 Cangjie/仓颉 SDK、华为 MAAS 集成与验证报告。
  - 结合鸿蒙生态目前 10 亿级设备、华为“天工计划”10 亿资金扶持，我们有机会成为**中国区 AI 记忆基础设施的事实标准**。

> 这里可以重点展示几份现有中文技术报告，让投资人感受到工程厚度，而不仅仅是概念。

---

### 5. 与竞品的差异（3–4 分钟）
- 海外：mem0 / Zep / Supermemory / 各种“Memory as a Service”。
  - 优点：API 友好、与 OpenAI/LangChain 生态契合。
  - 缺点：
    - 多数是“单一 Memory 类”，缺少原生多 Agent 架构。
    - 对中文 / 国内监管 / 本土生态支持弱，很难落地到金融、政企核心领域。
- 国内：云厂商内部记忆服务、传统知识库产品。
  - 通常作为“附属功能”，不是一个独立的基础设施层。
  - 与鸿蒙 / 仓颉 / 多云多模型的兼容性有限。
- **我们的定位**：
  - 在技术上：**以多智能体架构 + 图记忆为核心，做“下一代记忆内核”**。
  - 在市场上：**做“本土生态优先”的记忆基础设施层**，而不是某家云或某个大模型的附属品。

---

### 6. 典型场景与商业价值（3–4 分钟）
- **企业 Copilot / 智能客服**：
  - 记住每一个客户的全部历史交互，给出“量身定制”的回答与推荐。
  - 预期价值：减少人工工单 30%+，提升一次解决率，降低流失。
- **企业知识与流程中台**：
  - 把 IM 对话、文档、业务系统中的信息统一建模为记忆图。
  - 帮助企业构建“可提问、可追溯、可审计”的知识中台。
- **鸿蒙场景：个人/行业智能助手**：
  - 终端 + 云协同：在鸿蒙手机、平板、电脑、车机之间共享个人记忆。
  - 行业内：金融、教育、医疗等垂直场景的专用智能体。
- **多 Agent 协作系统（研究 & 工业结合）**：
  - 在 LOCOMO、MIRIX 等基准上提升效果，用论文+BenchMark 讲故事。

> 投资人更关心的是“有没有真实项目”、“客户愿意为什么买单”，这里可以尽量用数字和目标行业来讲。

### 7. 商业模式与市场空间（3–4 分钟）
- **商业模式**：
  1. 企业订阅 / License：按 QPS / 记忆量 / 实例数计费。
  2. 私有化项目 + 行业解决方案：与 ISV/SI 合作，交付 AI 记忆中台项目。
  3. 生态分成：与鸿蒙应用开发者、华为云市场进行调用量分成。
- **市场空间**：
  - AI 基础设施整体是千亿级美金市场，其中记忆层 100–200 亿美金空间。
  - 中国本土市场在“安全合规 + 本土生态”维度有独特需求，给了我们天然缺口。
